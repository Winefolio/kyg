# Improve Journey Wine Options & Fix Shop Finder

## Overview

Two issues with the learning journey chapter wine selection experience:

1. **Wine options too restrictive** — Currently 1 option per tier (budget/splurge). Users' local shops may not carry these specific wines. Need more alternatives per tier + a fallback "Ask Pierre" button for people intimidated by wine shops.
2. **"Find a wine shop nearby" broken** — Shows "Couldn't find wine shops right now" because `GOOGLE_PLACES_API_KEY` is likely missing from env config.

## Part 1: More Wine Options + Ask Pierre

### Problem

Each chapter has exactly 2 wine options (1 budget, 1 splurge) generated by `createWineOptions()` in `scripts/seed-journeys.ts`. If a user's local shop doesn't carry the specific wine, they're stuck. Many users are also intimidated going into wine shops and need help knowing what to say.

### Solution

**A. Expand to 2+ options per tier in seed data**

Update `createWineOptions()` to accept multiple budget/splurge options:

```typescript
// scripts/seed-journeys.ts
function createWineOptions(
  budgets: Array<{ desc: string; askFor: string; min: number; max: number; producers: string[] }>,
  splurges: Array<{ desc: string; askFor: string; producers: string[] }>
): WineOption[] {
  return [
    ...budgets.map(b => ({
      description: b.desc, askFor: b.askFor,
      priceRange: { min: b.min, max: b.max, currency: 'USD' as const },
      exampleProducers: b.producers, level: 'budget' as const
    })),
    ...splurges.map(s => ({
      description: s.desc, askFor: s.askFor,
      exampleProducers: s.producers, level: 'splurge' as const
    }))
  ];
}
```

Then update each chapter's wine options to include 2 budget + 1-2 splurge options. Example for Bordeaux:
- Budget 1: "Any Bordeaux Red" ($12-20)
- Budget 2: "Côtes de Bordeaux or Blaye" ($10-18)
- Splurge: "Médoc or Saint-Émilion" ($25-45)

**B. Update WineOptionsList layout**

The current 2-column grid works fine for 3-4 cards. No layout changes needed — just renders more cards.

**C. Add "Can't find these? Ask Pierre" button**

Add a button below the wine options in `JourneyDetail.tsx` that opens Pierre with chapter context pre-loaded:

```tsx
// JourneyDetail.tsx — below <NearbyWineShops />
<button onClick={() => {
  const msg = `I'm working on "${chapter.title}" from the ${journey.title} journey. ` +
    `I need a ${chapter.wineRequirements?.wineType || 'wine'} — ` +
    `specifically ${chapter.wineRequirements?.grapeVariety || chapter.wineRequirements?.region || 'something for this chapter'}. ` +
    `My local shop doesn't have the suggested options. Can you help me find an alternative?`;
  sessionStorage.setItem("pierre_initial_message", msg);
  // Trigger Pierre to open
  window.dispatchEvent(new CustomEvent('pierre:open'));
}}>
  Can't find these wines? Ask Pierre for help
</button>
```

The `SommelierFAB` listens for the `pierre:open` custom event, reads the initial message from sessionStorage, opens the chat sheet, and auto-sends it.

### Key Files

- `scripts/seed-journeys.ts` — Update `createWineOptions()` + all chapter wine data
- `client/src/pages/JourneyDetail.tsx:491` — Add "Ask Pierre" button below wine options
- `client/src/components/SommelierFAB.tsx` — Listen for `pierre:open` event
- `client/src/components/SommelierChatSheet.tsx` — Handle initial message auto-send
- `shared/schema.ts` — No changes (WineOption[] already supports multiple)

## Part 2: Fix Wine Shop Finder

### Problem

`NearbyWineShops` component calls `GET /api/wine-shops/nearby` which requires `GOOGLE_PLACES_API_KEY` env var. If missing, returns 503: "Wine shop search is not configured". This is why it shows "Couldn't find wine shops right now."

### Solution

1. **Check if `GOOGLE_PLACES_API_KEY` is set in Railway** — if not, add it
2. **Check local `.env`** — add the key for dev testing
3. **Improve the error message** — distinguish between "not configured" (hide the feature) vs "API error" (show retry)

### Key Files

- `server/routes/places.ts` — The API route (already implemented correctly)
- `client/src/components/NearbyWineShops.tsx` — Error handling UX
- `.env` / Railway env vars — Add `GOOGLE_PLACES_API_KEY`

## Risks & Gotchas

- **Re-seeding**: Need to re-run seed script against the DB, which may conflict with existing journey progress. Consider using `ON CONFLICT DO UPDATE` for chapters.
- **Pierre event pattern**: Using `CustomEvent` + `sessionStorage` is simple but slightly fragile. Alternative: React context. Recommend starting with the event approach — it's minimal and works.
- **Google Places API cost**: Already rate-limited (20 req/60s) and cached (30 min TTL). Costs are minimal for this usage.
